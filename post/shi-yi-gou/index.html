题目

校内OJ

分析

1.考点

树形dp

2.思路

显然“两点一线”性价比最高，剩下的无论在哪，只要加一条边就可以多连一个点（此时加任意一条边也只能多连一个点）。

然后，我就卡在定义上了...根据树形dp的尿性，dp[i]dp[i]dp[i]是以第iii个节点为根的子树的最多连接节点的数量，因此需要第二维代价。然而，非常合理的T了。所以只能是一维，那么必须换一个定义。

正确的定义需要利用我们最开始“瞪”出来的“两点一线”。首先用f[i]f[i]f[i]表示以第iii个节点为根的子树最多“两点一线“，发现状态无法转移，所以我们需要将其分成两类：与iii节点匹配的f[i]f[i]f[i]，不与iii节点匹配的g[i]g[i]g[i]。最终答案就为f[1]f[1]f[1]

g[i]g[i]g[i]显然是加f[k]f[k]f[k](kkk为iii子节点),f[i]f[i]f[i]则需要考虑将自己连上那个子节点（g[x]−f[to[i]]+g[to[i]]+1g[x] - f[to[i]] + g[to[i]] + 1g[x]−f[to[i]]+g[to[i]]+1最大即所有点都能构成“两点一线”，此时f[i]f[i]f[i]连任意节点都与其答案相同）。最后先考虑“两点一线”，若不够，则在加“一点一线”

代码

#include <cstdio>
#include <cstring>
#include <algorithm>
using namespace std;
const int MAXN = 1e5 + 5;
int head[MAXN], to[MAXN << 1], nxt[MAXN << 1], cnt, n, k, dp[MAXN], f[MAXN], T;
void add(int u, int v) {
    nxt[++cnt] = head[u];
    head[u] = cnt;
    to[cnt] = v;
}
void dfs(int x, int fa) {
    dp[x] = f[x] = 0;
    for (int i = head[x]; i; i = nxt[i]) {
        if (to[i] == fa) continue;
        dfs(to[i], x);
        dp[x] += dp[to[i]];
        f[x] += dp[to[i]];
    }
    for (int i = head[x]; i; i = nxt[i]) {
        if (to[i] == fa) continue;
        dp[x] = max(dp[x], f[x] - dp[to[i]] + f[to[i]] + 1);
    }
}
int main() {
    scanf("%d", &T);
    while (T--) {
        scanf("%d %d", &n, &k);
        for (int i = 2, u; i <= n; ++i) {
            scanf("%d", &u);
            add(i, u);
            add(u, i);
        }
        dfs(1, -1);
        if (k <= dp[1] * 2) printf("%d\n", (k + 1) / 2);
        else printf("%d\n", dp[1] + (k - dp[1] * 2));
        cnt = 0;
        memset(head, 0, sizeof(head));
    }
    return 0;
}
