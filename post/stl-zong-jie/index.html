一 STL

C++标准模版库

组成部分：

容器，算法，迭代器

Ⅰ. vector （动态数组）

定义：vector是一个能够存放任意类型的动态数组，能够增加 和压缩数据。

头文件： #include<vector>

定义方式： vector<数据类型> 名称

常用函数

Ⅱ. map （关联容器）

定义：map是一个关联容器，它提供一对一（其中第一个可以称为关键字，每个关键字只能在map中出现一次，第二个可能称为该关键字的值）的数据处理能力，由于这个特性，它完成有可能在我们处理一对一数据的时候，在编程上提供快速通道。

头文件： #include<map>

定义方式： map<键的类型, 数据类型> 名称

数据存储方式

mapStudent.insert(map<int,string>::value_type (3,"student_three"));//存入数据
map<int,string>::iterator iter; //迭代器
map<int,string>::reverse_iterator iter;//反向迭代器

Ⅲ. set （集合）

定义：set是一种容器，它所包含的元素的值是唯一的。

头文件：#include<cstdio

定义方式：set<数据类型> 名称

set<string>:: iterator s; //迭代器
se.insert("cantaloupes");//存入数据

Ⅳ. stack（栈）

定义：栈是一种数据结构，它按照后进先出的原则存储数据，先进入的数据被压入栈底，最后的数据在栈顶，需要读数据的时候从栈顶开始弹出数据。

头文件： #include<stack>

定义方式： stack<数据类型> 名称

常用函数

empty() //堆栈为空则返回真
pop() //移除栈顶元素（不会返回栈顶元素的值）
push() //在栈顶增加元素
size() //返回栈中元素数目
top() //返回栈顶元素

Ⅴ queue（队列）

定义：队列是一种特殊的线性表，是一种先进先出的数据结构。它只允许在前端进行删除操作，而在后端进行插入操作。进行插入操作的端称为队尾，进行删除操作的端称为队头。队列中没有元素时，称为空队列。

头文件： #include<queue>

定义方式： queue<数据类型> 名称

常用函数

q.empty()	//判断队列q是否为空，当队列q空时，返回true;否则为false（值为0(false)/1(true)）。
q.size()	//访问队列q中的元素个数。不可写成sizeof(q)或size(q)
q.push()	//会将一个元素a置入队列q中
q.front()	//返回队列q内的第一个元素(也就是第一个被置入的元素)。(不可写成front(q))
q.back();	//会返回队列q中最后一个元素(也就是最后被插入的元素)。(不可写成back(q))
q.pop()//会从队列q中移除第一个元素。(不可写成pop(q))

VI.priority_queue （优先队列）

定义：队列是一种先进先出的数据结构，在优先队列中，元素被赋予优先级。当访问元素时，最高优先级的元素最先访问。通常采用堆数据结构来实现。

头文件： #include<queue>

定义方式 :
priority_queue<数据类型> 名称 (升序)

priority_queue<int, vector, less > q (升序)

priority_queue<int, vector, greater > q (降序)

常用函数

push()	//将x加入优先队列
pop()	//队首元素出队
top()	//获得队首元素
empty()	//如果优先队列为空,返回true
size()	//返回priority_queue内元素的个数

以上所有东西使用时需加using namespace std;

二 图论

1.定义

图表示物件与物件之间的关系

2.术语

路径：至少两个相邻顶点组成的序列。
圈（环）：起点和终点重合的路径。
自环：一条边的起点与终点重合。
连通图：任意两点间都有路径连通。
权值：一个顶点到另一个顶点的花费，时间等。
无向边：没有指向的边。
无向图：所有的边都是无向边。
度：一个顶点所连接的边数。
树：没有环的无向连通图。
森林：没有环的无向非连通图。
有向边：有指向的边。
有向图：所有的边都是有向边。
出度：一个顶点到其他顶点的边数。
入度：其它顶点到一个顶点的边数。
度：一个顶点出度和入度的和。
有向环：起点与终点重合的一条有向路径。
有向无环图（DAG）：没有环的有向图。

2.存储方式：

1 邻接矩阵

上图储存为：

0 0 1 1
0 0 0 1
1 0 0 1
1 1 1 0

空间复杂度为 N2N^{2}N2,MLE警告
代码如下：

#include<cstdio>
int n, m;    // n为顶点数，m为边的数量
bool a[205][205], used[205];
int main(){
   scanf("%d %d", &n, &m);
   for(int i = 1; i <= m; ++i) {
   	int u, v;
   	scanf("%d %d", &u, &v);
   	a[u][v] = true;
   }
   return 0;
}
2 邻接表

上图储存为

1: 3 4
2: 4
3: 1 4
4: 1 2 3

空间复杂度为M，远小于邻接矩阵
代码如下：

#include <cstdio>
#include <vector>
using namespace std;
const int MAXN = 1005;
vector<int> G[MAXN];
int n, m;   
bool used[MAXN];
void addEdge(int u, int v){
	G[u].push_back(v);
}
int main(){
	scanf("%d %d", &n, &m);
	for(int i = 1; i <= m; ++i) {
		int u, v;
		scanf("%d %d", &u, &v);
		addEdge(u, v);
    }
	return 0;
}

三 DP 动态规划

定义

动态规划(dynamic programming)是运筹学的一个分支，是求解决策过程(decision process)最优化的数学方法。20世纪50年代初美国数学家R.E.Bellman等人在研究多阶段决策过程(multistep decision process)的优化问题时，提出了著名的最优化原理(principle of optimality)，把多阶段过程转化为一系列单阶段问题，利用各阶段之间的关系，逐个求解，创立了解决这类过程优化问题的新方法–动态规划。1957年出版了他的名著《Dynamic Programming》，这是该领域的第一本著作。

DP题满足的三大性质

1.最优子结构性质: 如果问题的最优解所包含的子问题的解也是最优的,我们就称该问题具有最优子结构性质(即满足最优化原理).最优子结构性质为动态规划算法解决问题提供了重要线索.

2.子问题重叠性质: 子问题重叠性质是指在用递归算法自顶向下对问题进行求解时,每次产生的子问题并不总是新问题,有些子问题会被重复计算多次.动态规划算法正是利用了这种子问题的重叠性质,对每一个子问题只计算一次,然后将其计算结果保存在一个表格中,当再次需要计算已经计算过的子问题时,只是在表格中简单地查看一下结果,从而获得较高的效率.

3.无后效性: 将各阶段按照一定的次序排列好之后,对于某个给定的阶段状态,它以前各阶段的状态无法直接影响它未来的决策,而只能通过当前的这个状态.换句话说,每个状态都是过去历史的一个完整总结.这就是无后向性,又称为无后效性.

步骤：

1.状态:最后一步与子问题.
2.确定状态转移方程.
3.确定边界.
4.计算顺序.

背包问题

背包问题是一种组合优化的NP完全问题。问题可以描述为：给定一组物品，每种物品都有自己的重量和价格，在限定的总重量内，我们如何选择，才能使得物品的总价格最高。

0/1背包

有N件物品和一个容量为V的背包。第i件物品的重量是w[i]，价值是v[i]。求解将哪些物品装入背包可使这些物品的重量总和不超过背包容量，且价值总和最大。

如图，第i行j列表示选取前i个物品装进容量为j的背包的最大价值，可得状态转移方程，见下代码

for (int i = 1; i <= N; ++i) {
    for (int j = 0; j <= V; ++j) {
        if(j >= w[i]) dp[i][j] = max(dp[i - 1][j - w[i]] + v[i], dp[i - 1][j]);
        else dp[i][j] = dp[i-1][j];
    }
}

完全背包

有N种物品和一个容量为V的背包，每种物品都有无限件可用。第i种物品的体积是c，价值是w。将哪些物品装入背包可使这些物品的体积总和不超过背包容量，且价值总和最大。
见代码

for (int i = 1; i <= n; ++ i)
    for (int j = c[i]; j <= v; ++ j)
        dp[j] = max(dp[j - c[i]] + w[i], dp[j]);

可以发现：与0/1背包相比，完全背包只是第二重循环的顺序发生了改变。我们保证用较小容量的最大价值去更新较大容量的最大价值，不重不漏。时间复杂度O（V*N），空间复杂度O（V）。

四 分治

定义：

分治，字面上的解释是“分而治之”，就是把一个复杂的问题分成两个或更多的相同或相似的子问题，再把子问题分成更小的子问题……直到最后子问题可以简单的直接求解，原问题的解即子问题的解的合并。

适用条件：

  1. 问题的规模缩小到一定的规模就可以较容易地解决。
  2. 问题可以分解为若干个规模较小的模式相同的子问题，即该问题具有最优子结构性质。
  3. 合并问题分解出的子问题的解可以得到问题的解。
  4. 问题所分解出的各个子问题之间是独立的，即子问题之间不存在公共的子问题。
二分查找

要求：数列有序
例题 ： 折半查找法

最初设左右端点为数组边界，中位数与需查找的数比较，缩近左右端点至中间，时间复杂度O(log(n)log_(n)log(​n));

#include <cstdio>
int n,a[100005],m,dp[1000005];
int main(){
	scanf("%d",&n);
	for (int i = 1 ; i  <= n ; i++){
		scanf("%d",&a[i]);
		dp[a[i]] = 1;
	} 
	scanf("%d",&m);
	if (!dp[m]){
		printf("-1");
		return 0;
	}
	int l = 1,r = n;
	while(l < r){
		int mid = (l + r) / 2;
		if (a[mid] < m) l = mid;
		if (a[mid] > m) r = mid;
		if (a[mid] == m){
			printf("%d",mid);
			return 0;
		}
	}
	
	return 0;
}
快速幂
ll qpow(ll a, ll n) {
    ll re = 1;
    while(n) {
        if(n & 1 == 1) re = (re * a) % MOD;
        n >>= 1;
        a = a * a % MOD;
    }
    return re % MOD;
}

五 树

定义：树是一种数据结构，它是由n（n>=0）个有限结点组成一个具有层次关系的集合。把它叫做“树”是因为它看起来像一棵倒挂的树，也就是说它是根朝上，而叶朝下的。

特点：

每个结点有零个或多个子结点；
没有父结点的结点称为根结点；
每一个非根结点有且只有一个父结点；
除了根结点外，每个子结点可以分为多个不相交的子树；

术语

节点的度：一个节点含有的子树的个数称为该节点的度；
叶节点：度为0的节点称为叶节点；
父节点：若一个节点含有子节点，则这个节点称为其子节点的父节点；
子节点：一个节点含有的子树的根节点称为该节点的子节点；
树的度：一棵树中，最大的节点的度称为树的度；
节点的层：从根开始定义起，根为第1层，根的子节点为第2层，以此类推；
树的高度：树中节点的最大层次；
森林：由m（m>=0）棵互不相交的树的集合称为森林；

查找树根

方法：根为没有父节点的节点，找到那个没有父节点的节点就行了

for(int i = 1, k, m = 0; i <= n; m = 0, ++i){
	if(tree[i].fa == 0) {
		root = i;
		break;
	}
}

二叉树

定义：每个节点最多含有两个子树的树称为二叉树；

遍历方式
  • 先序遍历：根，左子树，右子树
  • 中序遍历：左子树，根，右子树
  • 后序遍历：左子树，右子树，根
  • 层次遍历：BFS

六 并查集

题目：

在开始时让每个元素构成一个单元素的集合，然后按一定顺序将属于同一集合的元素所在的集合合并，其间要多次查找一个元素在哪个集合中。

做法

用树模拟集合，查找集合是否相同即为查找树根是否相同

初始化

将每个数的根初始化为自己

void MakeSet(int n) {
	for(int i = 1; i <= n; ++i) fa[i] = i;
	return ;
}

查找

int FindSet(int x) {
	while(x != fa[x]) {
		fa[x] = fa[fa[x]];
		x = fa[x];
	}
	return x;
}
路径压缩

在树深度高时，用时长，则通过将所有数的父节点设为根

int FindSet(int x) {
	if(x != fa[x])	return fa[x] = FindSet(fa[x]);
	else return x;
}

合并

void UnionSet(int x, int y) {
	int u = FindSet(x), v = FindSet(y);
	if(u == v) return ;
	fa[u] = v;
	return ;
}
